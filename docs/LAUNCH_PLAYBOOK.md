# Launch Playbook (Bilingual)

This playbook helps you launch Codex Bridge in English and Chinese channels in one week.

## 1. Positioning

- One-liner:
  - EN: `Remote-control your local Codex threads from Telegram in 2 minutes.`
  - ZH: `2 分钟把本机 Codex 远程装进 Telegram。`
- Core pain:
  - EN: Codex is desktop-first, but real work happens when you're away from your desk.
  - ZH: Codex 强在桌面，但真实工作经常发生在你离开电脑的时候。

## 2. Launch assets checklist

- App download link (latest release).
- 30-second demo GIF/video (pairing → `/threads` → first reply).
- 3 screenshots:
  - onboarding,
  - `/threads` binding in Telegram,
  - approval + final response.
- Press kit (ready-to-use): `assets/press-kit/`
- README link.
- Privacy note: anonymous metrics only; no message content collected.

## 3. Post templates

### Product Hunt / Hacker News (EN)

Title:
`Remote Codex from Telegram in 2 minutes (No CLI)`

Body:
`Codex Bridge lets you keep working on your local Codex thread from your phone. It supports thread-aware binding, remote approvals, queue status, and usage checks (/usage). Main setup path is no-CLI and takes about 2 minutes.`

CTA:

### 掘金 / 少数派 / 即刻（ZH）

正文：

CTA：
`项目地址：https://github.com/tonyHu08/CodeX_Bridge`

## 4. Weekly cadence

- Tuesday: English technical post + short demo.
- Thursday: Chinese tutorial post + practical case.
- Weekend: user story recap and feedback highlights.

## 5. Launch day checklist (30 min)

1. Make sure the latest Release has DMG + ZIP + screenshots in notes.
2. Update README first screen (value + screenshots + 2-minute setup).
3. Prepare 1 short clip and 3 screenshots from `assets/press-kit/`.
4. Post in 1 EN channel + 1 ZH channel the same day.
5. Stay online for 2 hours to answer comments (pairing failures, `/threads` tips).

## 6. Comment/reply templates

  - `For power users, self-hosting mode is available in docs.`
- ZH:

  - `高级用户可以在文档里开启自托管模式。`
